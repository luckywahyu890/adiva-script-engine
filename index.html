Pendahuluan

Filosofi desain ADIVA Script Engine

ADIVA Script Engine adalah engine game 2D sederhana yang dirancang agar mudah digunakan dan mudah dipahami. Fokus utama engine ini bukan pada sistem kompleks seperti engine besar, tetapi pada workflow yang cepat untuk membuat game berbasis gambar.

Berbeda dengan banyak engine modern yang menggunakan texture atlas besar untuk mengejar performa maksimum, ADIVA justru mendorong penggunaan gambar terpisah satu file untuk setiap animasi atau objek.

Pendekatan ini membuat proses pembuatan aset menjadi jauh lebih fleksibel, terutama jika menggunakan AI image generator. Kamu bisa membuat gambar karakter, musuh, atau pemandangan secara terpisah tanpa perlu menggabungkannya ke atlas terlebih dahulu.

Dengan workflow ini, proses pembuatan game menjadi lebih cepat dan eksperimental. Developer bisa fokus pada ide gameplay tanpa terlalu dibatasi oleh pipeline aset yang rumit.

01

Game Loop

GameLoop — inti dari semua game

Game loop adalah jantung dari setiap game. ADIVA menyediakan GameLoop yang mengelola eksekusi frame secara otomatis.

Kode di luar game.render() dieksekusi sekali saja saat halaman dimuat — cocok untuk load aset. Kode di dalam game.render() dieksekusi terus-menerus (~16.6ms sekali / 60fps).

setup
JavaScript 
// ambil elemen canvas
const canvas = document.getElementById("canvas")

// buat game loop
const game = new GameLoop(canvas)

// dieksekusi SEKALI saat halaman dimuat
console.log("init...")

game.render(dt => {
  // dieksekusi TERUS-MENERUS ~16.6ms
  console.log("frame!")
})
Parameter dt adalah delta time dalam detik. Gunakan ini untuk membuat gerakan konsisten di semua device.
02

Virtual Button

VirtualButton — kontrol sentuh untuk mobile

Buat tombol layar sentuh dengan new VirtualButton(). Cocok untuk game mobile. Posisi menggunakan properti left, right, top, bottom dalam pixel.

Properti Keterangan
text Label teks yang tampil di tombol
left / right Jarak dari kiri/kanan layar (px)
bottom / top Jarak dari bawah/atas layar (px)
onPress Callback saat tombol ditekan
onRelease Callback saat tombol dilepas
JavaScript 
const left = new VirtualButton({
  text: "<=",
  left: 20,
  bottom: 30,
  onPress: () => { left.press = true },
  onRelease: () => { left.press = false }
})

const right = new VirtualButton({
  text: "=>",
  left: 100,
  bottom: 30,
  onPress: () => { right.press = true },
  onRelease: () => { right.press = false }
})

game.render(dt => {
  if (right.press) {
    console.log("kanan")
  } else if (left.press) {
    console.log("kiri")
  }
})
03

Load Gambar

Aset — load texture sebelum game mulai

Sebelum menampilkan gambar, semua texture harus di-load terlebih dahulu menggunakan Aset. Gunakan pattern *N untuk load beberapa frame animasi sekaligus.

Wildcard *N — Run*3.png artinya load Run1.png, Run2.png, Run3.png secara otomatis.
JavaScript 
const gl = game.gl
const renderer = new Renderer(gl, canvas)

const listImg = {
  run:  { path: "./Art/Run*3.png" },  // 3 frame animasi
  wall: { path: "./Art/Wall*1.png" }, // 1 frame
}

const aset = new Aset(gl, listImg)
let number = 0

async function init() {
  // tunggu sampai semua gambar selesai dimuat
  const texture = await aset.load()

  game.render(dt => {
    // game berjalan di sini
    
    number ++
    renderer.test(number)
    
  })
}

init()
Selalu gunakan async/await saat load aset. Tanpa ini, gambar bisa hitam karena belum selesai dimuat.
04

Tampilkan Gambar

Draw — render entitas ke layar

Semua entitas (player, tembok, dll) menggunakan format object of object. Setiap entitas memiliki properti posisi, ukuran, dan gambar.

properti entitas
x / y
posisi
Koordinat di dunia game
width / height
ukuran
Lebar dan tinggi sprite (px)
img
animasi
Key nama dari listImg
speed
kecepatan
Kecepatan pergerakan
JavaScript 
const players = {
  clientA: { x: 150, y: 150, width: 100, height: 100, speed: 300 }
}

const walls = {
  A: { x: 0, y: 400, width: 100, height: 100, img: "wall" }
}

const drawPlayer = new Draw(players, texture, renderer)
const drawWall   = new Draw(walls, texture, renderer)

game.render(dt => {
  drawPlayer.update(dt)
  drawWall.update(dt)

  for (let id in players) {
    let p = players[id]
    p.img = "run"

    if (right.press) p.x += 5
    else if (left.press) p.x -= 5
  }
})
05

Kamera

cameraOn — kamera mengikuti pemain

Gunakan game.cameraOn() untuk membuat kamera mengikuti entitas tertentu. Setelah itu, kirimkan game.camera ke semua pemanggilan draw.update().

JavaScript 
game.render(dt => {
  // kamera mengikuti players.clientA
  game.cameraOn(players, "clientA")

  // teruskan game.camera ke setiap Draw
  drawPlayer.update(dt, game.camera)
  drawWall.update(dt, game.camera)
})
Entitas UI (HUD, tombol, skor) tidak perlu menerima game.camera agar tetap di posisi layar.
06

Game State

Collision.gameState — kelola scene aktif

Game state memungkinkan kamu beralih antar scene seperti menu → play → level2 → shop. Setiap entitas harus memiliki properti scene yang sesuai.

gameState.add()
"play"
Aktifkan state tertentu
scene
"play"
Properti wajib di tiap entitas
JavaScript 
const physics = new Collision(150)
physics.gameState.add("play") // aktifkan state "play"

const players = {
  clientA: {
    x: 150, y: 150, width: 100, height: 100,
    scene: "play",  // wajib ada!
    speed: 300
  }
}

// walls dibuat dengan loop
const walls = {}
for (let i = 0; i < 10; i++) {
  walls[i] = {
    x: 100 * i, y: 400, width: 100, height: 100,
    type: "walls", img: "wall", scene: "play"
  }
}

// kirim physics.gameState ke Draw
const drawPlayer = new Draw(players, texture, renderer, physics.gameState)
const drawWall   = new Draw(walls, texture, renderer, physics.gameState)
Untuk mengganti scene, cukup ubah physics.gameState.add("menu"). Semua entitas yang memiliki scene: "play" akan otomatis tidak dirender. Penjelasan lebih detail tentang sistem scene akan dibahas pada STEP terakhir.
07

Collision & Gravity

Physics — tabrakan dan gaya gravitasi

ADIVA menggunakan sistem physics berbasis velocity. Gerakan dilakukan lewat vx/vy, bukan mengubah posisi langsung.

properti physics
Properti Keterangan
vx Kecepatan horizontal (px/detik)
vy Kecepatan vertikal (px/detik)
gravity Gaya tarik ke bawah per detik
speed Kecepatan maksimal player
useCollider Aktifkan collider box custom
colliderBox Ukuran hitbox — biasanya lebih kecil dari sprite
urutan eksekusi
JavaScript 
const physics = new Collision(150)
physics.gameState.add("play")
physics.grid.buildStatic(walls) // daftarkan tembok ke grid

const players = {
  clientA: {
    x: 150, y: 150, width: 100, height: 100,
    scene: "play", speed: 300,
    gravity: 200, vx: 0, vy: 0,
    useCollider: true,
    colliderBox: { width: 40, height: 100 }
  }
}

game.render(dt => {
  game.cameraOn(players, "clientA")
  drawPlayer.update(dt, game.camera)
  drawWall.update(dt, game.camera)

  for (let id in players) {
    let p = players[id]

    // 1. set velocity berdasarkan input
    if (right.press)      p.vx =  p.speed
    else if (left.press)  p.vx = -p.speed
    else                  p.vx =  0

    // 2. terapkan physics & collision
    physics.applyPhysics(dt, p)

    // 3. tambahkan gravitasi
    p.vy += p.gravity * dt
  }
})
08

Jump & Platformer

platformerLogic — lompat, double jump, animasi otomatis

ADIVA menyediakan physics.platformerLogic() yang menangani semua logika platformer secara otomatis: jump buffering, double jump, deteksi ground, dan state animasi.

maxJump
2
Jumlah lompatan (1 = normal, 2 = double jump)
jumpPower
450
Kekuatan dorongan ke atas
jumpBufferTimer
auto
Set dari onPress tombol jump
gravity
1000
Tinggi-rendah lompatan
tombol jump
JavaScript 
const jump = new VirtualButton({
  text: "(^)",
  right: 40,
  bottom: 80,
  onPress: () => {
    const p = players.clientA
    p.jumpBufferTimer = p.jumpBufferMax
  }
})
urutan eksekusi (penting!)
JavaScript 
const players = {
  clientA: {
    x: 150, y: 150, width: 100, height: 100,
    scene: "play", speed: 300,
    gravity: 1000, vx: 0, vy: 0,
    useCollider: true, colliderBox: { width: 40, height: 100 },
    maxJump: 2, jumpPower: 450
  }
}

game.render(dt => {
  game.cameraOn(players, "clientA")
  drawPlayer.update(dt, game.camera)
  drawWall.update(dt, game.camera)

  for (let id in players) {
    let p = players[id]

    if (right.press)      p.vx =  p.speed
    else if (left.press)  p.vx = -p.speed
    else                  p.vx =  0

    physics.applyPhysics(dt, p)    // ① collision dulu
    physics.platformerLogic(dt, p) // ② logic lompat
    p.vy += p.gravity * dt           // ③ gravitasi terakhir
  }
})
Urutan applyPhysics → platformerLogic → gravity tidak boleh dibalik. Urutan ini menentukan apakah jump terdeteksi dengan benar.
09

Animasi Otomatis

baseAnim — state animasi dari platformerLogic

Setelah physics.platformerLogic() berjalan, properti p.state pada entitas player akan diisi otomatis sesuai kondisi fisika saat itu — tanpa perlu kamu atur secara manual.

Yang perlu kamu lakukan hanyalah memetakan p.state ke nama animasi lewat objek baseAnim, lalu assign ke p.img. Hapus semua baris p.img = "run" yang statis dari step sebelumnya.

state yang dihasilkan otomatis
idle → run → jump → fall
p.state Kondisi
idle Di tanah, tidak bergerak horizontal (vx = 0)
run Di tanah, bergerak ke kiri atau kanan
jump Di udara, bergerak ke atas (vy < 0)
fall Di udara, bergerak ke bawah (vy > 0)
setup listImg & baseAnim

Daftarkan semua animasi di listImg — termasuk idle, jump, dan fall yang sebelumnya belum ada. Kemudian buat objek baseAnim sebagai peta dari nama state ke key animasi.

JavaScript 
const listImg = {
  run:  { path: "./Art/Run*3.png"  },
  idle: { path: "./Art/Idle*3.png" },
  jump: { path: "./Art/Jump*2.png" },
  fall: { path: "./Art/Fall*2.png" },
  wall: { path: "./Art/Walll*1.png" },
}

// peta state → key animasi
const baseAnim = {
  idle: "idle",
  run:  "run",
  jump: "jump",
  fall: "fall",
}
Key di baseAnim harus sama persis dengan nilai p.state yang dihasilkan platformerLogic. Value-nya adalah key dari listImg.
pakai di game loop

Di dalam loop, hapus p.img = "run" yang statis. Gantikan dengan satu baris yang membaca p.state secara dinamis lewat baseAnim.

JavaScript 
game.render(dt => {
  game.cameraOn(players, "clientA")
  drawPlayer.update(dt, game.camera)
  drawWall.update(dt, game.camera)

  for (let id in players) {
    let p = players[id]

    if (right.press)      p.vx =  p.speed
    else if (left.press)  p.vx = -p.speed
    else                  p.vx =  0

    physics.applyPhysics(dt, p)    // ① collision
    physics.platformerLogic(dt, p) // ② p.state diisi otomatis
    p.vy += p.gravity * dt           // ③ gravitasi

    // ④ terapkan animasi berdasarkan state otomatis
    p.img = baseAnim[p.state]
  }
})
Satu baris ajaib: p.img = baseAnim[p.state] — inilah inti dari step ini. platformerLogic mengisi p.state, kamu hanya perlu membacanya.
perubahan dari step 8
Dihapus
p.img = "run"
Baris statis ini diganti sepenuhnya — tidak relevan lagi
Ditambahkan
baseAnim

Ditambahkan

10

Deteksi Objek (AABB)

physics.aabb() — mendeteksi tabrakan antar entitas

Pada step ini kita akan memahami physics.aabb() untuk mendeteksi apakah dua objek bertabrakan.

Contoh paling umum adalah player mengambil koin. Ketika player menyentuh koin, kita bisa menjalankan kode tertentu seperti menambah skor atau menghapus koin.

konsep utama
Fungsi Keterangan
physics.aabb() Mendeteksi tabrakan antar entitas
type Penanda jenis entitas (coin, enemy, item, dll)
buildDynamic() Mendaftarkan objek yang bisa bergerak atau berubah
AABB berarti Axis-Aligned Bounding Box — metode collision sederhana menggunakan kotak persegi.
membuat objek koin

Kita buat objek koin seperti entitas lain, tetapi wajib memiliki properti type. Properti ini digunakan oleh physics.aabb() untuk mengenali objek.

JavaScript 
const coin = {
  coin1: {
    x: 250,
    y: 200,
    width: 100,
    height: 100,

    // type digunakan untuk physics.aabb()
    type: "coin",

    img: "coin",
    scene: "play"
  }
}
daftarkan ke physics grid

Objek seperti koin harus didaftarkan ke physics.grid.buildDynamic() di setiap frame.

JavaScript 
game.render(dt => {

  // WAJIB di awal render
  physics.grid.buildDynamic(coin)

  game.cameraOn(players, "clientA")

  drawPlayer.update(dt, game.camera)
  drawWall.update(dt, game.camera)
  drawCoin.update(dt, game.camera)

})
buildDynamic() harus dipanggil setiap frame. Jika tidak, collision tidak akan terdeteksi.
deteksi tabrakan player dan koin
JavaScript 
physics.aabb(
  players,
  "coin", // type yang ingin dicek
  (g1, g2) => {

    // g1 = player
    // g2 = objek yang bertabrakan (coin)

    renderer.test(`${g2.type} berhasil diambil`)
    // maka akan muncul tulisan berwarna hijau
    //  "coin berhasil diambil"
  }
)
alur collision
player → menyentuh → coin → callback()
Setelah koin terdeteksi, kamu bisa langsung menjalankan logika seperti menambah skor, memainkan suara, atau menghapus koin.
ringkasan step 10
  • Setiap entitas interaktif harus memiliki type.
  • Daftarkan objek dinamis ke physics.grid.buildDynamic().
  • Gunakan physics.aabb() untuk mendeteksi tabrakan.
  • Gunakan callback (g1, g2) untuk menjalankan logika saat collision.
11

Memutar Audio

Aset.play() — memainkan sound effect di game

Pada step ini kita akan belajar cara memainkan audio di dalam game.

Audio biasanya digunakan untuk memberikan feedback kepada pemain, seperti ketika mengambil koin, melompat, atau menyerang musuh.

konsep utama
Fungsi Keterangan
Aset Class yang mengelola gambar dan audio
listAudio Daftar file audio yang akan dimuat
aset.play() Memainkan audio berdasarkan nama
utils.timer() Mengatur jeda agar audio tidak diputar terlalu cepat
Sistem audio ADIVA menggunakan nama key untuk memanggil suara. Jadi kita cukup memanggil nama audio tanpa perlu membuat objek audio baru setiap kali.
membuat daftar audio

Pertama kita buat daftar audio yang ingin digunakan di game.

JavaScript 
const listAudio = {
  coin: "./Audio/Coin.mp3"
}
memasukkan audio ke class Aset

Audio dimasukkan ke parameter kedua saat membuat objek Aset.

JavaScript 
const aset = new Aset(gl, listImg, listAudio)
Pastikan nama audio di listAudio sama dengan nama yang dipanggil di aset.play().
memutar audio

Untuk memainkan suara, gunakan fungsi aset.play().

JavaScript 
aset.play("coin")
audio saat player mengambil koin

Kita bisa memutar audio ketika player bertabrakan dengan koin menggunakan physics.aabb().

JavaScript 
physics.aabb(
  players,
  "coin",
  (g1, g2) => {

    renderer.test(`${g2.type} berhasil diambil`)

    utils.timer(dt, "s", 0.3, () => {
      aset.play("coin")
    })

  }
)
utils.timer() digunakan agar suara tidak diputar terlalu cepat setiap frame. Dengan jeda ini audio terdengar lebih natural.
alur audio di game
player → menyentuh → coin → aset.play()
ringkasan step 11
  • Buat daftar audio di listAudio.
  • Masukkan daftar audio ke class Aset.
  • Gunakan aset.play() untuk memutar suara.
  • Gunakan utils.timer() agar audio tidak diputar setiap frame.
12

Membuat Level 2

Berpindah dari scene play ke play2 menggunakan flag

Pada step ini kita akan membuat level kedua.

Ketika player menyentuh flag (bendera), maka game akan berpindah dari scene play ke scene play2.

Pada step ini juga ada beberapa perubahan penting:

  • Menambahkan gambar flag.
  • Menambahkan gambar wall baru untuk level 2.
  • Membuat iterasi wall level kedua.
  • Membuat object flag sebagai pemicu level.
  • Mengubah gameState dari play ke play2.
konsep utama
Konsep Keterangan
scene Menentukan object muncul pada level tertentu
gameState Menyimpan scene yang sedang aktif
flag Object pemicu perpindahan level
physics.aabb() Mendeteksi tabrakan player dengan flag
Sistem scene ADIVA bekerja dengan cara memfilter object berdasarkan property scene. Jika scene object sama dengan gameState aktif, maka object tersebut akan dirender.
menambahkan gambar baru

Pada step ini kita menambahkan dua gambar baru:

  • flag → penanda akhir level
  • wallB → tembok khusus untuk level kedua
JavaScript 
const listImg = {

  run : {path : "./Art/Run*3.png"},
  idle : {path : "./Art/Idle*3.png"},
  jump : {path : "./Art/Jump*2.png"},
  fall : {path : "./Art/Fall*2.png"},

  coin : {path : "./Art/Coin*1.png"},
  wall : {path : "./Art/Wall*1.png"},

  flag : {path : "./Art/Flag*1.png"},
  wallB : {path : "./Art/WallB*1.png"}

}
membuat wall level 2

Level kedua juga memiliki tembok sendiri.

Kita membuatnya menggunakan perulangan (loop) seperti level pertama, tetapi dengan scene : "play2".

JavaScript 
for(let i = 0; i < 10; i++){

  walls[i + "level2"] = {
    x: 100 * i,
    y: 300,
    width: 100,
    height: 100,
    type :"walls",
    img : "wallB",
    scene: "play2"
  }

}
Perbedaan utama level 1 dan level 2 hanya pada scene dan gambar tembok.
membuat object flag

Flag berfungsi sebagai penanda akhir level.

JavaScript 
const flag = {

  flag1 : {
    x: 900,
    y: 200,
    width: 100,
    height: 100,
    type :"flag",
    img : "flag",
    scene: "play"
  }

}
menampilkan flag

Flag harus dirender menggunakan class Draw.

JavaScript 
const drawFlag = new Draw(flag, texture, renderer, physics.gameState)

Lalu update di dalam game loop.

JavaScript 
drawFlag.update(dt, game.camera)
flag sebagai object dynamic

Agar collision bisa mendeteksi flag, kita harus menambahkannya ke buildDynamic().

JavaScript 
physics.grid.buildDynamic(coin, flag)
deteksi player menyentuh flag

Ketika player menyentuh flag, kita akan:

  • Mengubah scene dari play ke play2
  • Mengubah scene player
  • Mereset posisi player
JavaScript 
physics.aabb(players, "flag", (g1, g2)=>{

  physics.gameState.delete("play")
  physics.gameState.add("play2")

  g1.scene = "play2"

  g1.x = g1.originalX
  g1.y = g1.originalY

})
alur perpindahan level
player → menyentuh → flag → play2
ringkasan step 12
  • Menambahkan gambar flag dan wallB.
  • Membuat wall baru untuk level kedua.
  • Membuat object flag sebagai penanda akhir level.
  • Mendeteksi tabrakan menggunakan physics.aabb().
  • Mengubah scene dari play ke play2.
